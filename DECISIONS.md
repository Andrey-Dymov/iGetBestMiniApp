## 0. Референсное приложение

**iGetBestMiniApp разрабатывается по образцу приложения iGetBest.**

| Параметр | Значение |
|----------|----------|
| Референс | iGetBest (Swift/SwiftUI) |
| Путь | `/Users/smokebook13/Documents/Development/iGetBest` |
| Назначение | Поддержка принятия решений (Multi-Criteria Decision Making) |

### Описание iGetBest

- **Сравнения** — задачи выбора между вариантами по критериям
- **Варианты** — альтернативы для сравнения (с изображениями, URL)
- **Параметры** — критерии оценки с весами 0–10
- **Критерии** — шкала оценок для каждого параметра (числовые и текстовые)
- **Значения** — матрица вариант×параметр, расчёт totalScore и позиций
- **Экраны** — список сравнений, результаты с медалями, блоки параметров, индикаторы, графики

*Подробнее: `iGetBest/DecisionMaking_README.md`, `Описание экранов.md`*

### Разработка и проверка

| Этап | Где | Как |
|------|-----|-----|
| **Разработка** | Локально | `npm run dev` → localhost:5173 |
| **Проверка в Telegram** | GitHub Pages | Деплой → открыть Mini App в боте |
| **Отладка** | Локально | DevTools, hot reload |

**Workflow:** Пишем локально → при необходимости деплоим → проверяем в Telegram. Не нужно деплоить каждый раз — только когда нужно проверить в реальном Mini App.

### Telegram-специфичные функции локально

| Функция | Локально (localhost) | В Telegram |
|---------|----------------------|------------|
| **initData** (авторизация) | `mockTelegramEnv()` — подмена тестовыми данными | Реальные данные пользователя |
| **Cloud Storage** | Fallback на localStorage (при mock) | Реальное хранилище Telegram |
| **ThemeParams** | `mockTelegramEnv()` — подмена темы | Тема из Telegram |
| **Haptic** | Игнорируется (нет поддержки в браузере) | Работает на мобильных |

**Варианты локальной проверки:**

1. **Mock** — `mockTelegramEnv()` из SDK: initData, ThemeParams, fallback Cloud Storage → localStorage. UI и логика без деплоя.
2. **Test Environment** — Telegram Test Server: бот может открывать `http://localhost` или IP. Полная проверка в реальном Telegram.
3. **Деплой на Pages** — production-проверка.

*Рекомендация: Mock для повседневной разработки; Test Environment или деплой — для проверки Cloud Storage и авторизации.*

### Компактный формат — что это

Текстовый (не JSON) формат для быстрого импорта. Можно скопировать из чата или заметок и вставить.  
**Основной формат обмена между пользователями — JSON.** Компактный — для удобства вставки.

**Пример:**
```
Сравнение: Выбор мотоцикла
Параметры: Цена (1200000:7-1600000:4-1800000:3), вес 8, Комфорт (Неудобно:2-Нормально:6-Комфортно:10), вес 7
Kawasaki ZX-10R: 1600000, 160, Комфортно.
Honda CBR1000RR: 1200000, 180, Нормально.
```
